Service connection

1. What is Azure DevOps used for in software development?

Correct Option: C) To manage the entire software development lifecycle

Solution and Explanation:
Azure DevOps provides an integrated set of tools to support planning, development, testing, delivery, and monitoring of software applications. It is used to manage the entire software development lifecycle efficiently.




2. Which of the following services is NOT a feature of Azure DevOps?

Correct Option: D) Virtual Machine management

Solution and Explanation:
Azure DevOps includes services like Boards, Pipelines, Repos, Test Plans, and Artifacts. It does not directly manage virtual machines; that is handled through Azure Portal or Azure Resource Manager.




3. Which Azure DevOps service is used for version control?

Correct Option: B) Azure Repos

Solution and Explanation:
Azure Repos provides Git repositories or Team Foundation Version Control (TFVC) for source control of your code. It helps in managing code versions and collaboration between team members.




4. What is the function of Azure Pipelines?

Correct Option: A) To automate build and release processes

Solution and Explanation:
Azure Pipelines automates the building, testing, and deployment of code projects. It supports CI/CD practices and can work with any language or platform.




5. What is the use of Azure Boards in Azure DevOps?

Correct Option: C) To track work items like bugs, tasks, and user stories

Solution and Explanation:
Azure Boards is used to plan, track, and discuss work across teams. It provides Kanban boards, backlogs, dashboards, and custom reporting for effective project tracking.




6. Which of the following is a type of service connection in Azure DevOps?

Correct Option: A) Azure Resource Manager

Solution and Explanation:
Azure Resource Manager (ARM) is a common type of service connection used to deploy resources to Azure directly from Azure DevOps pipelines.




7. What is required when setting up a GitHub service connection in Azure DevOps?

Correct Option: C) A personal access token (PAT) from GitHub

Solution and Explanation:
To connect Azure DevOps with GitHub, you need a GitHub personal access token (PAT) that authorizes Azure DevOps to access repositories securely.




8. Which service connection is typically used to connect to container registries?

Correct Option: B) Docker Registry

Solution and Explanation:
The Docker Registry service connection allows Azure DevOps to authenticate and interact with container registries like Docker Hub or Azure Container Registry for pushing and pulling container images.




9. Which authentication method is commonly used with Azure Resource Manager service connections?

Correct Option: A) Service principal with client secret

Solution and Explanation:
The most common authentication method for ARM connections is using a service principal with a client ID and secret, granting limited and secure access to Azure resources.




10. What is the main purpose of service connections in Azure DevOps?

Correct Option: D) To securely connect Azure DevOps to external and Azure services

Solution and Explanation:
Service connections in Azure DevOps enable secure communication between Azure DevOps and external systems (like Azure, GitHub, Docker Hub) to execute CI/CD tasks seamlessly.




11. Where do you navigate in Azure DevOps to create a new service connection?

Correct Option: B) Project Settings > Service connections

Solution and Explanation:
In Azure DevOps, service connections are managed under the "Project Settings" section. You can create and configure new connections under "Service connections".




12. Which role is required to create a service connection in Azure DevOps?

Correct Option: C) Project Administrator

Solution and Explanation:
To create and manage service connections, a user must have the "Project Administrator" role, which grants the necessary permissions within the project.




13. What is typically required when creating an Azure Resource Manager service connection?

Correct Option: A) Subscription ID and Service Principal credentials

Solution and Explanation:
To connect Azure DevOps with Azure via ARM, you need the Azure Subscription ID and a service principal (client ID, secret, and tenant ID) for authentication.




14. What setting allows you to restrict who can use a service connection?

Correct Option: D) Security settings of the service connection

Solution and Explanation:
The security settings of a service connection allow you to manage permissions such as who can use, edit, or administer the service connection.




15. Which option must be enabled to grant Azure DevOps access to manage resources in the Azure subscription when creating a service connection?

Correct Option: B) Grant access permission to all pipelines

Solution and Explanation:
The option "Grant access permission to all pipelines" ensures that all pipelines in the project can use the created service connection without additional configuration.




16. What is the primary purpose of an Azure Resource Manager (ARM) service connection?

Correct Option: A) To allow Azure DevOps to deploy resources to an Azure subscription

Solution and Explanation:
An ARM service connection enables Azure DevOps to authenticate with an Azure subscription and deploy or manage resources through pipelines or release processes.




17. What authentication method is commonly used for ARM service connections?

Correct Option: B) Service Principal (automatic or manual)

Solution and Explanation:
ARM service connections typically use a Service Principal to authenticate. Azure DevOps can create one automatically or you can configure it manually.




18. Which of the following permissions are required by the Service Principal for ARM service connections?

Correct Option: D) Contributor or higher on the target subscription or resource group

Solution and Explanation:
The Service Principal used in an ARM service connection should have Contributor permissions or above to deploy and manage resources in the selected scope.




19. What scope can you choose while setting up an ARM service connection?

Correct Option: C) Subscription or Resource Group

Solution and Explanation:
When creating an ARM service connection, you can scope it to an entire subscription or a specific resource group depending on the required access control level.




20. What is the benefit of using a managed identity for an ARM service connection?

Correct Option: B) No credentials need to be stored in Azure DevOps

Solution and Explanation:
A managed identity eliminates the need to store sensitive credentials (like secrets or client IDs), enhancing security while enabling access to Azure resources.




21. What is a Service Principal in the context of Azure authentication?

Correct Option: C) An identity used by applications or services to access Azure resources

Solution and Explanation:
A Service Principal is an identity created for use with applications, hosted services, and automated tools to access Azure resources securely.




22. What components are required to manually configure a Service Principal?

Correct Option: D) Client ID, Client Secret, and Tenant ID

Solution and Explanation:
When manually configuring a Service Principal, you need the Client ID (App ID), Client Secret (password), and Tenant ID to authenticate securely with Azure.




23. What role should a Service Principal have in a subscription to allow deployments?

Correct Option: B) Contributor

Solution and Explanation:
The Contributor role provides the necessary permissions to create and manage all types of Azure resources, which is typically required for deployment scenarios.




24. How is a Service Principal different from a user account?

Correct Option: A) It is intended for use by applications and automation, not human users

Solution and Explanation:
Service Principals are non-interactive Azure AD identities that are used by applications or services, while user accounts are designed for human interaction.




25. Which Azure CLI command is used to create a Service Principal?

Correct Option: C) az ad sp create-for-rbac

Solution and Explanation:
The az ad sp create-for-rbac command is used to create a new Service Principal and assign it a role-based access control (RBAC) role in Azure.




26. What is a Managed Identity in Azure?

Correct Option: A) An automatically managed identity for applications to access Azure resources securely

Solution and Explanation:
A Managed Identity is a feature of Azure Active Directory that provides automatically managed identities for applications and services to authenticate to Azure resources securely without storing credentials in code.




27. Which types of Managed Identities are available in Azure?

Correct Option: B) System-assigned and User-assigned

Solution and Explanation:
Azure supports two types of Managed Identities: system-assigned (tied to the lifecycle of the resource) and user-assigned (independent of the resource).




28. What is a key benefit of using Managed Identity in a service connection?

Correct Option: C) Eliminates the need to manage credentials

Solution and Explanation:
Managed Identity handles the credentials automatically, removing the risk and overhead of managing secrets, keys, or passwords in code or configuration.




29. How is a system-assigned managed identity tied to its Azure resource?

Correct Option: A) It is automatically created and deleted with the resource

Solution and Explanation:
A system-assigned identity is tied directly to the lifecycle of the Azure resource. When the resource is deleted, the identity is also removed.




30. Which role should a managed identity be granted to allow access to a storage account?

Correct Option: D) Storage Blob Data Contributor

Solution and Explanation:
To allow a managed identity to access and modify blob data in a storage account, it needs the Storage Blob Data Contributor role assigned at the appropriate scope.




31. What is the main purpose of using a service connection in a pipeline?

Correct Option: B) To securely connect the pipeline to external services or Azure resources

Solution and Explanation:
A service connection allows a pipeline to securely authenticate and connect to external resources like Azure, GitHub, Docker, etc., without hardcoding credentials.




32. Which YAML key is used to define a service connection in an Azure DevOps pipeline?

Correct Option: C) connection

Solution and Explanation:
In Azure DevOps YAML pipelines, the `connection` keyword is used under steps like `AzureCLI@2` or `AzureRmWebAppDeployment@4` to specify the service connection to use.




33. In which section of the Azure DevOps UI can you view and manage service connections?

Correct Option: A) Project Settings > Service Connections

Solution and Explanation:
In Azure DevOps, service connections are managed under "Project Settings" in the "Service Connections" tab. Here you can create, modify, and delete connections.




34. What happens if a pipeline uses a service connection without having permission?

Solution and Explanation:
If a pipeline does not have permission to use a service connection, it will fail during execution, typically with an authorization error message.




35. How can you grant a pipeline access to use a service connection?

Correct Option: B) Add the pipelines security group to the service connection permissions

Solution and Explanation:
To allow a pipeline to use a service connection, you must give the pipeline's identity or security group permission under the service connection's "Security" settings.




36. What is the purpose of approval checks in service connections?

Correct Option: A) To ensure manual verification before using a service connection

Solution and Explanation:
Approval checks provide an extra layer of control by requiring manual approval before a pipeline can use a particular service connection, enhancing security and accountability.




37. Where can you configure approval checks for a service connection?

Solution and Explanation:
You can configure manual approval requirements under the Approvals and checks tab for each service connection, defining who must approve its usage in a pipeline.




38. What role must a user have to approve a service connection usage if approval is enabled?

Solution and Explanation:
Only users who are explicitly added to the approval list for that service connection can approve its use in a pipeline run.




39. How can auditing be enabled for service connection usage?

Azure DevOps provides audit logs that track usage of service connections, including approvals and changes, accessible from the organization-level settings.




40. Why is auditing important for service connections in Azure DevOps?

Solution and Explanation:
Auditing tracks when, how, and by whom a service connection was used or modified, supporting compliance and early detection of misuse or unauthorized changes.




41. What is a recommended practice when storing secrets in service connections?

Azure Key Vault provides secure storage for secrets and integrates with service connections, enhancing the security of sensitive credentials.

43. What is the benefit of using managed identity over client secrets?

Managed identities offer secure authentication without storing or rotating secrets, reducing the risk of credential leaks and improving maintainability.
