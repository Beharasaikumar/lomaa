1. What differentiates AWS Guard Duty from AWS WAF?

Solution and Explanation:
Guard Duty is for network security, whereas WAF is for application security.

AWS Shield Advanced provides detailed attack diagnostics to enhance security.

6. What feature of AWS WAF can help with bot protection?

Rate limiting helps prevent bot attacks by restricting request rates.

16. How does AWS WAF enable protection against SQL injection attacks?

Solution and Explanation:

17. What is a DDoS (Distributed Denial of Service) attack?

Solution and Explanation:
A DDoS attack is an attack where multiple systems flood the target with traffic, causing service disruption.




18. How does AWS WAF use geo-location blocking?

Solution and Explanation:
AWS WAF can filter traffic based on geographic location by using rules that block traffic from specific countries.




19. What is whitelisting?

Solution and Explanation:
Whitelisting is a security technique where only specific IP addresses or commands are allowed to access a system.




20. What feature helps in bot protection?

Solution and Explanation:
Features such as rate limiting, blocking suspicious traffic, and using CAPTCHA can help protect against bots.




21. How does AWS WAF protect against Cross-Site Scripting (XSS) attacks?

Solution and Explanation:
AWS WAF protects against XSS by defining rules to filter out malicious scripts in HTTP requests.




22. What is SAST (Static Application Security Testing)?

Solution and Explanation:
SAST analyzes source code for vulnerabilities without executing the program, detecting issues early in the development lifecycle.




23. Which of the following is a popular SAST tool?

Solution and Explanation:
A popular SAST tool is OWASP ZAP, which helps in identifying vulnerabilities in web applications.




24. How does Selenium work in SAST?

Solution and Explanation:
Selenium is primarily used for automated testing of web applications, not specifically for SAST. It can be used in conjunction with other tools for security testing.




25. What type of vulnerabilities can SAST typically find?

Solution and Explanation:
SAST tools can typically find vulnerabilities like Cross-Site Scripting (XSS), SQL Injection, and buffer overflows by analyzing the source code.




26. What is the typical way to fix vulnerabilities identified by SAST?

Solution and Explanation:
Typically, vulnerabilities identified by SAST are fixed by manually reviewing the code and applying patches.




27. What is DAST (Dynamic Application Security Testing)?

Solution and Explanation:
DAST is a method for identifying vulnerabilities in a running application by testing its behavior and interactions.




28. Which of the following is a DAST tool?

Solution and Explanation:
A popular DAST tool is OWASP ZAP, which helps in identifying vulnerabilities in running applications.




29. How does DAST find vulnerabilities?

Solution and Explanation:
DAST identifies vulnerabilities by executing the application and observing its behavior in real-time.




30. Which of the following vulnerabilities can DAST typically identify?

Solution and Explanation:
DAST tools can typically identify vulnerabilities such as SQL Injection, Cross-Site Scripting (XSS), and Command Injection.




31. When should DAST testing be performed?

Solution and Explanation:
DAST testing should be performed during application runtime to detect vulnerabilities in a running application.

38. What type of vulnerabilities can be detected using Grype?

40. What is the primary purpose of penetration testing?

41. What is the primary purpose of AWS Inspector?

43. How does AWS WAF help in mitigating DDoS attacks?

44. Which of the following is a type of vulnerability commonly detected by penetration testing?

Solution and Explanation:

45. What does a vulnerability scanner do?

Solution and Explanation:

46. What is the function of a Web Application Firewall (WAF)?

Solution and Explanation:
WAF filters and monitors HTTP traffic to and from a web application to protect it from common exploits.




47. What is the purpose of using an Intrusion Detection System (IDS)?

Solution and Explanation:
An IDS is used to detect unauthorized access and potential threats in a network by monitoring traffic and activities.




48. What is the difference between Static and Dynamic Application Security Testing (SAST and DAST)?

Solution and Explanation:
SAST tests the source code, while DAST tests a running application to identify vulnerabilities.




49. What is the primary goal of fuzz testing?

Solution and Explanation:
Fuzz testing aims to identify and exploit potential vulnerabilities in an application by inputting random or unexpected data.




50. Which of the following tools can be used for fuzz testing?

AFL (American Fuzzy Lop) is a popular tool for fuzz testing applications by generating random test inputs.




51. How can you mitigate Cross-Site Scripting (XSS) attacks in a web application?

52. Which of the following is the main benefit of using AWS Shield?

53. How does a Security Information and Event Management (SIEM) system help with threat detection?

A SIEM system helps by collecting and analyzing security event data to detect potential threats.




54. What is the purpose of an API Gateway in cloud security?

55. Which of the following can be used to secure API endpoints?

56. What is a zero-day vulnerability?

Solution and Explanation:
A zero-day vulnerability refers to a security flaw that has no available patch or fix at the time of discovery.




57. What type of attack involves exploiting a vulnerability in a web application to run arbitrary code on a server?

Solution and Explanation:
Remote Code Execution (RCE) involves exploiting a vulnerability to execute arbitrary code on a server.




58. What is the primary purpose of using encryption in transit?

Solution and Explanation:
Encryption in transit protects data from unauthorized access while being transferred over the network.




59. Which of the following security measures can protect against SQL Injection attacks?

Solution and Explanation:
Using prepared statements with parameterized queries helps prevent SQL Injection by ensuring proper handling of input.




60. What does the concept of least privilege mean in security?

Solution and Explanation:
Least privilege means giving users only the minimum level of access they need to perform their tasks.
