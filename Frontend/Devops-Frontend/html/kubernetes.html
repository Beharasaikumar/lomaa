KUBERNETES

1. Which of the following is a core component of Kubernetes?

Solution and Explanation:
The Kubernetes API Server is a core component responsible for handling REST requests and cluster management.




2. What is the primary difference between Docker Swarm and Kubernetes?

Solution and Explanation:
Kubernetes provides a full suite of features beyond container orchestration, whereas Docker Swarm focuses primarily on orchestration.




3. What are the main components of a Deployment Controller?

Solution and Explanation:
Deployment Controllers primarily consist of ReplicaSets and Pods to manage application deployment and scaling.




4. What is the purpose of a ReplicaSet in Kubernetes?

Solution and Explanation:
A ReplicaSet ensures that a specified number of pod replicas are running at any given time.




5. What is a Pod in Kubernetes?

Solution and Explanation:
A Pod is a set of containers that share the same network namespace and storage.


6. Why would you use a StatefulSet in Kubernetes?

Solution and Explanation:
StatefulSets are used for applications requiring stable network identities and persistent storage.




7. What is a use case for a DaemonSet?

Solution and Explanation:
DaemonSets ensure that a specific Pod runs on every node, useful for monitoring agents.




8. How can you monitor logs in Kubernetes?

Solution and Explanation:
Logs can be monitored using the 'kubectl logs' command or the Kubernetes dashboard.




9. What does troubleshooting in Kubernetes typically involve?

Solution and Explanation:
Troubleshooting often involves reviewing pod logs and events to diagnose issues.




10. What are liveness, readiness, and startup probes used for?

Solution and Explanation:
These probes are used to monitor and manage pod health and readiness.


11. What is the difference between Horizontal Pod Autoscaler (HPA) and Vertical Pod Autoscaler (VPA)?

Solution and Explanation:
HPA adjusts the number of pods, while VPA adjusts the resource requests and limits of pods.




12. What is the process of upgrading a Kubernetes cluster called?

Solution and Explanation:
The process of upgrading a Kubernetes cluster is called Cluster Upgrade.




13. What is a multi-container pod?

Solution and Explanation:
A multi-container pod is a pod containing containers that share storage and network resources.




14. Which deployment strategy involves gradually rolling out changes?

Solution and Explanation:
A Rolling Update strategy involves gradually rolling out changes.




15. What is the purpose of an Ingress Controller in Kubernetes?

Solution and Explanation:
The Ingress Controller manages external access to services in the cluster.




16. How do you map an Ingress resource to a service?

Solution and Explanation:
You map an Ingress resource to a service by specifying the service name and port in the Ingress configuration.




17. What is the difference between a volume and a volume mount?

Solution and Explanation:
A volume is a storage resource, while a volume mount is the link to the container.




18. What is the difference between a Persistent Volume (PV) and a Persistent Volume Claim (PVC)?

Solution and Explanation:
A PV is a storage resource, while a PVC is a request for storage.




19. What does Node Affinity refer to in Kubernetes?

Solution and Explanation:
Node Affinity refers to restrictions on where Pods can run based on node labels.




20. What are taints and tolerations used for?

Solution and Explanation:
Taints and tolerations are used to control which Pods can be scheduled on nodes.




21. What is a ResourceQuota in Kubernetes?

Solution and Explanation:
A ResourceQuota is a limit on the number of resources a namespace can consume.




22. What is the role of the kube-scheduler?

Solution and Explanation:
The kube-scheduler is responsible for scheduling Pods onto nodes.




23. What is the purpose of the Kubernetes Proxy?

Solution and Explanation:
The Kubernetes Proxy is responsible for routing traffic to the appropriate services.




24. What does the Control Manager do in Kubernetes?

Solution and Explanation:
The Control Manager is responsible for controlling the replication of Pods.




25. What is etcd in a Kubernetes cluster?

Solution and Explanation:
etcd is a distributed key-value store used for configuration data in Kubernetes.




26. How do master nodes communicate with worker nodes?

Solution and Explanation:
Master nodes communicate with worker nodes through the Kubernetes API server.




27. What is dynamic provisioning of storage?

Solution and Explanation:
Dynamic provisioning automatically creates storage volumes based on demand.




28. What is the difference between manual and dynamic provisioning of storage?

Solution and Explanation:
Manual provisioning requires pre-defined volumes, while dynamic provisioning creates volumes based on demand.




29. What happens when a pod is evicted?

Solution and Explanation:
When a pod is evicted, it is either moved to another node or restarted/rescheduled.




30. What does it mean when a pod is killed?

Solution and Explanation:
When a pod is killed, it was intentionally terminated by the system or user.




31. What does it indicate when a node is not ready?

Solution and Explanation:
When a node is not ready, it usually means the node is unreachable or experiencing issues.




32. What happens when a pod crashes?

Solution and Explanation:
When a pod crashes, it is automatically restarted by Kubernetes.




33. What are startup probes used for?

Solution and Explanation:
Startup probes are used to determine if an application has successfully started.




34. How do you define resource limits for a Pod?

Solution and Explanation:
Resource limits for a Pod are defined under the "resources" section in the pod specification.




35. What is the function of a Service in Kubernetes?

Solution and Explanation:
The function of a Service in Kubernetes is to provide external access to Pods.




36. What is a Helm chart?

Solution and Explanation:
A Helm chart is a package manager for Kubernetes applications, helping you define, install, and upgrade applications.




37. What is the purpose of a ConfigMap?

Solution and Explanation:
A ConfigMap is used to store non-confidential configuration data that can be consumed by applications running in Pods.




38. What does a StatefulSet manage in Kubernetes?

Solution and Explanation:
A StatefulSet manages applications that require stable identities and persistent storage.




39. What is the function of a DaemonSet?

Solution and Explanation:
A DaemonSet ensures that a copy of a pod runs on all or some nodes in the cluster.




40. How is a Kubernetes cluster scaled?

Solution and Explanation:
A Kubernetes cluster can be scaled by adding more nodes, adjusting resource limits, or changing the pod replicas in deployments.




41. What are the main roles of the Kubernetes API?

Solution and Explanation:
The Kubernetes API is responsible for providing a REST interface for managing resources and enabling communication with various components of the Kubernetes cluster.




42. What does the term "container orchestration" refer to?

Solution and Explanation:
Container orchestration refers to managing the lifecycle and deployment of containers across a cluster, ensuring that applications run smoothly and scale as needed.




43. What is the function of Kubernetes namespaces?

Solution and Explanation:
Kubernetes namespaces are used to isolate resources within a cluster, allowing multiple teams or applications to coexist in the same cluster while being isolated from each other.




44. What does the term "kubelet" refer to?

Solution and Explanation:
The kubelet is the primary agent that runs on each node in a Kubernetes cluster, ensuring that containers are running and healthy according to the specifications in the pod.




45. What is the purpose of the Kubernetes Scheduler?

Solution and Explanation:
The Kubernetes Scheduler is responsible for assigning Pods to nodes based on resource availability, ensuring that the Pods are scheduled efficiently across the cluster.




46. What is a Kubernetes deployment?

Solution and Explanation:
A Kubernetes deployment is used to manage the lifecycle of replicated Pods, ensuring that the desired number of Pods are running and scaled accordingly.




47. What is the command to view the status of Pods in a namespace?

Solution and Explanation:
The command `kubectl get pods -n ` is used to view the status of Pods in a specific namespace.




48. How do you scale a deployment in Kubernetes?

Solution and Explanation:
To scale a deployment in Kubernetes, you can adjust the number of replicas in the deployment specification or use the `kubectl scale` command.




49. What command is used to delete a pod?

Solution and Explanation:
The command `kubectl delete pod ` is used to delete a pod in Kubernetes.




50. What is a Kubernetes ingress?

Solution and Explanation:
A Kubernetes ingress is a resource used for managing external access to services, typically HTTP, and provides routing and load balancing for services within the cluster.




51. How can Prometheus and Grafana be configured to send alerts to a specific user when their applications CPU usage exceeds 80%?

Solution and Explanation:
Set up an alert rule in Prometheus and configure Grafana notification channels to alert users based on specified thresholds.




52. What steps should you take to ensure that your secrets are encrypted using KMS in a Kubernetes deployment?

Solution and Explanation:
Use the Kubernetes Secrets API with KMS integration to ensure that your secrets are encrypted automatically.




53. In what scenario would you decide to use Horizontal Pod Autoscaler (HPA) over Vertical Pod Autoscaler (VPA) for scaling an application?

Solution and Explanation:
HPA is best for scaling based on traffic by adjusting the number of replicas, whereas VPA adjusts resource requests for existing pods.




54. Describe a scenario where cluster autoscaling is beneficial for resource utilization.

Solution and Explanation:
Cluster autoscaling allows for the addition of new nodes when demand spikes, ensuring resources are efficiently allocated.




55. How does dynamic provisioning using storage classes simplify storage management in a Kubernetes cluster?

Solution and Explanation:
Dynamic provisioning automates the creation of persistent volumes based on demand, making storage management more efficient.




56. If an application experiences a pod failure, what strategies would you implement to ensure data persistence and prevent data loss?

Solution and Explanation:
Implement persistent volumes with backup policies to ensure that data is retained and not lost in case of a pod failure.




57. After deploying a new version of an application, users report issues. How would you perform a rollback using Helm charts?

Solution and Explanation:
Use the command `helm rollback` to roll back to the previous version of the application deployed using Helm.




58. Explain a situation where a blue-green deployment strategy would be preferable over other deployment strategies.

Solution and Explanation:
Blue-green deployments are beneficial when minimizing downtime is critical during the application update.




59. What are the key advantages of using a canary release for a microservice update?

Solution and Explanation:
Canary releases allow for testing in a production environment with minimal risk by exposing updates to only a small subset of users initially.




60. Describe a scenario where a rolling update is beneficial and how it differs from a recreate deployment.

Solution and Explanation:
A rolling update allows for zero downtime deployment, whereas a recreate deployment will cause downtime since it involves replacing all pods at once.




61. How would you implement a DaemonSet to collect logs from all nodes in your Kubernetes cluster?

Solution and Explanation:
A DaemonSet ensures that a log collection agent is running on all nodes in the cluster. You can create a DaemonSet with a logging agent container to collect logs from each node.




62. When should you choose a Load Balancer service type over NodePort for your frontend application?

Solution and Explanation:
You should choose a Load Balancer service type when you need external access to your application without exposing the node IPs directly.




63. What scenarios would lead you to use a ClusterIP service instead of NodePort or Load Balancer?

Solution and Explanation:
A ClusterIP service is used when the service only needs to be accessed within the Kubernetes cluster. It doesn't expose the service to external traffic.




64. What are the default port ranges for services in Kubernetes, and why is it important to be aware of them?

Solution and Explanation:
The default port range for services in Kubernetes is 30000-32767. This range is used to avoid conflicts with other applications that may use lower numbered ports.




65. In what scenarios would you use hostPath volumes for your microservices, and what are the potential risks?

Solution and Explanation:
HostPath volumes are used when accessing the host's file system is required. The main risk is that it can lead to data loss if not managed carefully.




66. How would you configure RBAC at the namespace level to restrict access to sensitive resources?

Solution and Explanation:
To restrict access to sensitive resources at the namespace level, you should create Roles and RoleBindings specific to users. This allows fine-grained access control within the namespace.




67. What are the implications of setting RBAC permissions at the cluster level versus at the namespace level?

Solution and Explanation:
Cluster-level permissions are broader and apply to all namespaces, whereas namespace-level permissions are specific to that namespace and are more granular.




68. What are the benefits of running a self-managed EKS cluster compared to a standard Kubernetes cluster?

Solution and Explanation:
A self-managed EKS cluster gives you full control over the environment, but requires more responsibility for configuration and maintenance. It provides flexibility in resource allocation and management.




69. What challenges might you face when managing a self-hosted Kubernetes cluster?

Solution and Explanation:
Managing a self-hosted cluster can be challenging due to the increased operational complexity and maintenance required. This includes ensuring updates, monitoring, and troubleshooting are managed effectively.




70. How do you determine the appropriate resource requests and limits for your application pods?

Solution and Explanation:
You should analyze application performance metrics to determine appropriate resource requests and limits. This helps ensure that your pods are allocated the right resources for optimal performance.
