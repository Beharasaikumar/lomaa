Deployment of ML Models

1. What is the primary goal of deploying a machine learning model?

Correct Option: A) To make the model accessible for predictions and integrate it into production systems.

Solution and Explanation:
Model deployment aims to make the trained machine learning model available for real-time or batch predictions in a production environment. This allows the model to integrate with business processes and deliver insights to users.

**Steps to Deploy a Model**:
1. Choose a deployment environment (cloud, on-premise, etc.).
2. Set up an API or interface for model interaction.
3. Monitor and update the model in production as needed.




2. What is a common method for serving a machine learning model in production?

Correct Option: B) Using an API to serve model predictions in real-time to applications.

Solution and Explanation:
APIs are commonly used to expose machine learning models in production. These APIs allow external applications to send data to the model and receive predictions in real-time, making it easy to integrate the model into operational systems.

**Steps to Serve a Model Using an API**:
1. Create a REST API or gRPC service.
2. Deploy the model to the server.
3. Set up the API to receive data and send back predictions.




3. Which is a common challenge when deploying machine learning models to production?

Correct Option: C) Model drift and the need for regular updates to the model.

Solution and Explanation:
In production, models can experience "drift" where their predictions become less accurate over time due to changes in underlying data patterns. Regular updates and retraining are necessary to maintain the model's effectiveness.

**Common Challenges in Model Deployment**:
1. Model drift.
2. Scaling the model to handle large volumes of requests.
3. Ensuring the model's performance in different environments.

7. Which deployment strategy involves rolling out the new model to a small subset of users before full deployment?

Correct Option: B) Canary deployment.

Solution and Explanation:
Canary deployment is a strategy where the new model version is first released to a small group of users (the "canaries") before being made available to everyone. This helps catch potential issues before full-scale deployment.

**Steps for Canary Deployment**:
1. Select a small group of users.
2. Deploy the new model to this group.
3. Monitor performance and feedback before full deployment.




8. What is the main advantage of using a Blue-Green deployment strategy?

Correct Option: D) It allows for easy rollback to the previous version if the new model causes issues in production.

Solution and Explanation:
Blue-Green deployment minimizes downtime by maintaining two identical environments: one (Blue) is live, while the other (Green) is where the new model is deployed. If issues occur, traffic can easily be switched back to the Blue environment.

**Steps for Blue-Green Deployment**:
1. Set up two environments: Blue (current) and Green (new).
2. Deploy the new model to the Green environment.
3. Switch traffic to Green; if issues arise, switch back to Blue.




9. What is the key difference between a rolling deployment and a Blue-Green deployment?

Correct Option: C) A rolling deployment gradually replaces the old model version with the new version, while Blue-Green involves switching traffic between two environments.

Solution and Explanation:
Rolling deployment updates the model version incrementally, replacing the old version gradually. In contrast, Blue-Green deployment uses two environments, with one serving traffic while the other is updated.

**Key Characteristics of Each Strategy**:
- **Rolling deployment**: Gradual update, no downtime, but requires monitoring.
- **Blue-Green deployment**: Zero downtime, easy rollback, but requires two environments.

21. What is the main difference between containerization and virtualization?

Correct Option: A) Containerization runs applications in isolated environments using shared OS resources, while virtualization runs entire virtual machines with their own operating systems.

Solution and Explanation:

**Key Differences**:
1. Containers share the host OS kernel.

3. Containers are lightweight and faster to start.
